# dagster_prod

## Инструкция по локальному запуску

1. Убедитесь, что установлен Docker и `docker-compose`.
2. Создайте файл `.env` в корне проекта (запросите у DevOps или используйте тестовые пароли).
3. Запустите всю инфраструктуру одной командой:
   ```bash
   docker-compose up -d --build
   ```
4. **Dagster UI** (мониторинг и запуск дагов): http://localhost:3000
5. **MinIO UI** (просмотр сохраненных DataFrame): http://localhost:9001

Все новые ассеты нужно добавлять в папку `dags/assets/`.
